## CH.2 커머스 과제

---

## Step 1

### 구현 내용
1. Main 및 Product 클래스 생성
2. Product 클래스 상품명, 가격, 설명을 저장할 필드를 작성 
3. Main 클래스에서 Product 객체 생성 
4. 반복문을 사용해 상품 목록을 출력 
5. 0을 입력하면 프로그램이 종료되도록 구현

### 구현 의도
- 상품 정보를 하나의 객체로 관리하기 위해 Product 클래스 생성
- 여러 상품을 저장하기 위해 List를 사용
- Step 1에서는 상품 목록 출력과 종료 기능처럼 가장 기본적인 흐름을 먼저 구현했습니다.

---

## Step 2

### 구현 내용
1. CommerceSystem 클래스 생성
2. 프로그램 실행 흐름을 Main이 아닌 CommerceSystem에서 담당하도록 수정
3. Main에서는 데이터 생성과 실행만 담당하도록 구조 변경
4. start() 메서드를 통해 전체 실행 흐름이 시작되도록 구현

### 구현 의도
- step 1에서 Main이 담당하던 실행 흐름을 CommerceSystem 클래스로 분리
- Main은 프로그램 시작만 담당하고 실제 흐름은 CommerceSystem이 담당하도록 정리
- Step 2에서는 역할을 나누는 구조에 집중했습니다.

---

## Step 3

### 구현 내용
1. Category 클래스 생성
2. Category 클래스에서는 카테고리 이름과 Product 리스트를 관리하도록 필드 생성
3. 카테고리 목록 출력, 카테고리 선택, 상품 목록 출력, 상품 선택 기능 구현
4. 선택한 상품의 상세 정보를 출력하도록 구현

### 구현 의도
- Step 3에서는 카테고리별로 상품을 관리하고 조회할 수 있도록 구조를 확장했습니다.

---

## Step 4

### 구현 내용
- 각 클래스 필드를 private으로 변경하고 캡슐화를 적용하여 직접 접근하지 못하도록 설정

### 구현 의도
- getter를 사용해 데이터를 더 안전하게 관리하도록 수정했습니다.

## Lv 1

### 구현 내용
1. Cart 클래스를 생성하여 장바구니 기능을 분리
2. 상품을 장바구니에 추가할 수 있도록 구현
3. 장바구니에 담긴 상품 목록을 출력할 수 있도록 구현
4. 장바구니에 담긴 상품들의 총 주문 금액을 계산하도록 구현
5. 장바구니 확인 후 주문을 확정할 수 있도록 구현
6. 주문 완료 시 상품 재고가 차감되도록 구현
7. 주문 취소 시 장바구니를 비우도록 구현
8. Commercesystem은 전체 흐름 제어를 담당하고, Category, Product, Cart가 각각의 역할을 담당하도록 구조 정리
9. Data 클래스를 생성해 초기 상품 및 카테고리 데이터를 분리

### 구현 의도
- 장바구니와 주문 기능을 별도 클래스로 분리하여 역할을 나누고자 했습니다.
- 상품 선택부터 장바구니 확인, 주문 확정, 주문 취소까지 하나의 흐름으로 연결되도록 구현했습니다.
- 주문 확정 시 실제로 재고가 감소하도록 해서 단순 조회가 아닌 주문 기능까지 동작하도록 구성했습니다.
- Main과 CommerceSystem에 몰려 있던 코드를 정리하고, 각 클래스가 자신의 역할을 담당하도록 구조를 개선했습니다.
- 초기 데이터를 Data 클래스로 분리하여 Main을 더 간단하게 정리했습니다.

## Lv 2

### 구현 내용
1. 관리자 모드 진입 기능 추가
2. 관리자 비밀번호를 입력받아 인증하는 기능 구현
3. 비밀번호를 3회 이상 틀릴 경우 메인 메뉴로 돌아가도록 구현
4. 관리자 메뉴에서 상품 추가, 상품 수정, 상품 삭제, 전체 상품 현황 조회 기능을 선택할 수 있도록 구현
5. 상품 추가 시 카테고리르 선택한 뒤 상품명, 가격, 설명, 재고를 입력받아 새 상품을 등록할 수 있도록 구현
6. 상품 수정 시 상품명을 입력해 기존 상품을 찾고, 가격 / 설명 / 재고수량 중 원하는 항목만 수정할 수 있도록 구현
7. 상품 삭제 시 상품명을 입력해 해당 상품을 찾은 뒤 삭제 여부를 확인하고 삭제할 수 있도록 구현
8. 전체 상품 현황 기능을 통해 카테고리별 상품 정보를 한 번에 조회할 수 있도록 구현
9. 관리자 기능에서 잘못된 입력이 들어오는 경우 `Exception`을 사용해 예외처리하도록 구현했습니다.

### 구현 의도
- 일반 사용자 기능과 관리자 기능을 구분해 관리할 수 있도록 관리자 모드를 추가
- 관리자 기능은 상품 뎅치터를 직접 수정하거나 삭제할 수 있기 때문에, 비밀번호 인증을 통해 접근을 제한하도록 구현
- 상품 추가, 수정, 삭제, 조회 기능을 통해 관리자 입장에서 상품 정보를 관리할 수 있도록 구성
- 잘못된 입력으로 인해 프로그램이 종료되지 않도록 예외처리를 적용해 안정적으로 동작하도록 구현
